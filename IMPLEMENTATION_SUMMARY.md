### LOGIC SYSTEM

#### Rank Requirements
```
Bronze: 0+ points
Silver: 600+ points
Gold: 1200+ points
Platinum: 1800+ points
Diamond: 2400+ points
Master: 3200+ points
Grandmaster: 4000+ points
```

#### Rank-Locked Item Logic
1. **Cannot Be Purchased** - Rank-locked frames are earned, not bought
2. **Auto-Equip Option** - Can be manually equipped when rank achieved
3. **Auto-Unequip on Rank Loss** - Automatically reverts to basic frame if rank drops
4. **Persistent Ownership** - Once earned, always in inventory (but cannot equip if rank lost)

#### Auto-Unequip Flow
1. Player loses ranked match
2. Rank points decrease
3. System checks if rank dropped
4. If equipped frame requires higher rank:
   - Auto-unequips rank-locked frame
   - Reverts to basic frame
   - Updates Firestore
   - Shows notification (optional)
5. When rank regained, can re-equip

## 6. TECHNICAL NOTES FOR PERFORMANCE & IMPLEMENTATION

### Performance Optimizations

#### GPU Acceleration
- All animations use `transform`, `opacity`, `filter` (GPU-accelerated)
- Added `will-change` hints for animated properties
- Use `translateZ(0)` for hardware acceleration
- Avoid `width`, `height`, `top`, `left` (CPU-intensive)

#### Mobile Optimizations
```javascript
// Automatic particle reduction on mobile
@media (max-width: 768px) {
  .effect-fire-particles::before {
    display: none; // Reduce particle count
  }
}

// Device performance detection
const devicePerformance = detectDevicePerformance();
// Returns: 'low', 'medium', or 'high'

// Adaptive particle counts
low: 50% of particles
medium: 75% of particles
high: 100% of particles
```

#### Battery Efficiency
- Animation durations: 2-4s typical (not too frequent)
- Avoid simultaneous heavy animations
- Use CSS animations over JavaScript (more efficient)
- Respect `prefers-reduced-motion` for accessibility

#### Memory Management
- Reuse effect classes (don't create new elements)
- Clean up effects on component unmount
- Use CSS keyframes (browser-optimized)
- Avoid memory leaks with proper cleanup
